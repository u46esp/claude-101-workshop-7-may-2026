Talks on TDD

Three talks.
One obsession.

Getting Started

TDD as a Coding Design Tool

Most people arrive at TDD as a testing discipline. I wanted to reframe it entirely — as a design tool. When you treat tests as a way to think through structure before you write it, TDD stops feeling like overhead and starts feeling like the clearest way to think. This talk was my attempt to convince newcomers to make that mental shift first, before anything else.

Test Writing

How to Write Tests in TDD

Seeing tests named "works correctly" drove me a little mad. It tells you nothing about what the class is supposed to do. This talk was about rewiring how engineers write test names — not to verify after the fact, but to declare intention as you shape the class.

Advanced TDD

Outside-In & Inside-Out: Both at Once

The long-running debate: start from the outer boundary and work inward, or build from the core outward? My answer: both mindsets have a place, and the real skill is knowing which to reach for. This talk explored growing tests and code with confidence, no matter which direction you're moving.
